Contexte et enjeux

Le marché de la mode représente plus de 1 700 milliards de dollars mondialement. Pourtant, une étude McKinsey révèle que 60 % des vêtements achetés ne sont jamais portés. Le manque de visibilité sur sa garde-robe, la difficulté à créer des tenues cohérentes et l'absence d'organisation au quotidien engendrent achats superflus, gaspillage et frustration.

ClosetMixer répond à ce constat en proposant un assistant vestimentaire personnel numérique : photographier ses articles, générer des tenues adaptées à la météo et à l'occasion, planifier ses looks dans un calendrier et préparer ses valises de voyage.

🌍
Dimension multiculturelle différenciante L'application intègre nativement des styles culturels variés (mode modeste, K-Fashion, J-Fashion, traditionnel) et supporte 8 langues dont l'arabe (RTL). Ce positionnement vise une audience internationale souvent ignorée par les acteurs du marché.

Enjeux clés

  • Enjeu usage : simplifier la prise de décision vestimentaire quotidienne
  • Enjeu consommation : réduire les achats inutiles en valorisant l'existant
  • Enjeu culturel : proposer une application inclusive pour toutes les identités vestimentaires
  • Enjeu business : constituer une base utilisateurs qualifiée pour de futures fonctionnalités premium

Description détaillée des écrans

E-01 — Garde-robe

Écran d'accueil de l'application. Il affiche l'ensemble des articles sous forme de grille à 2 colonnes avec un ratio d'image 3:4 (portrait), conforme au standard des applications de mode.

  • En-tête : photo de profil cliquable + titre "Closet Mixer" (police serif dorée) + bouton d'ajout
  • Chips de filtre : "Tout", Vêtement, Chaussure, Bijou, Maquillage, Couvre-chef, Accessoire, Skincare
  • Carte article : photo plein cadre, bouton favori (cœur) en superposition, étiquette sous-catégorie et couleur sous la carte
  • Suppression : appui long sur une carte → dialogue de confirmation → suppression définitive (F-18)
  • État vide : icône + message d'invitation à ajouter le premier article

E-02 — Ajouter un article

Formulaire d'ajout accessible depuis l'icône "+" de la garde-robe. Conforme à la spec Cm-mob_002 : tous les sélecteurs sont des listes déroulantes (ExposedDropdownMenuBox), non des boutons.

  • Photo : zone cliquable 240dp avec sélecteur de galerie système
  • Catégorie : liste déroulante (7 catégories)
  • Sous-catégorie : liste filtrée dynamiquement selon la catégorie choisie
  • Couleur : palette de 28 pastilles colorées avec validation par coche
  • Métal : affiché uniquement si catégorie = Bijou (Or, Argent, Rosé, Fantaisie, Aucun)
  • Style culturel : liste déroulante (Modest, K-Fashion, J-Fashion, Traditionnel, Neutre)
  • Bouton Enregistrer : activé uniquement si la sous-catégorie est renseignée

E-03 — Tenues (Génération)

Écran de génération de tenues combinant météo locale, style culturel et occasion.

  • Bandeau météo : température + description + emoji météo, fond or translucide
  • Chips style culturel : Neutre, Modeste, K-Fashion, J-Fashion, Traditionnel (scroll horizontal)
  • Chips occasion : Travail, Casual, Soirée, Voyage, Sport
  • Grille asymétrique "Today's Mix" : haut (7/12 largeur) + chaussure/bijou empilés (5/12), bas pleine largeur
  • Auto-génération : une tenue est générée automatiquement à l'ouverture de l'écran (F-20)
  • Bouton "GÉNÉRER UNE TENUE" : déclenche l'algorithme de régénération, indicateur de chargement
  • Partage : icône Partager dans la barre du haut, visible après génération — ouvre le sélecteur système Android avec photos + description textuelle (hashtags inclus) (F-16)
  • Logique météo : si température < 10°C, privilégier bottes/baskets ; si style = Modest, inclure couvre-chef
  • Harmonie métal/bijou : tons chauds dans la tenue → préférence or/rosé ; tons froids → préférence argent ; sinon aléatoire (F-13)
  • Label bijou enrichi : l'étiquette de la carte bijou affiche la sous-catégorie ET le métal (ex : "COLLIER · OR")

E-04 — Calendrier

Vue mensuelle permettant de visualiser et planifier les articles jour par jour.

  • En-tête : titre "Calendrier" + navigation mensuelle avec flèches < > (mois précédent / suivant)
  • Grille 7 colonnes : Dim–Lun–Mar–Mer–Jeu–Ven–Sam, décalage automatique du 1er du mois, reconstruit à chaque changement de mois
  • Indicateurs : point doré sous les jours avec un article planifié, fond or pâle sur le jour sélectionné, couleur primaire pour aujourd'hui
  • Panel jour sélectionné : s'affiche sous la grille au tap sur un jour
    • Si un article est planifié : icône ✓ + libellé "Tenue planifiée" + bouton "Modifier"
    • Si aucun article : bouton "Planifier une tenue" pleine largeur
  • Sélecteur d'article (ModalBottomSheet) : liste les articles de la garde-robe avec sous-catégorie, catégorie et couleur — tap sur un article l'affecte au jour sélectionné
  • Résumé mensuel : carte affichant le nombre de jours planifiés ce mois

E-05 — Voyages

Gestion des déplacements et des valises virtuelles. L'écran fonctionne en deux vues : liste des voyages et détail d'un voyage (valise).

Vue liste

  • Cartes voyage : nom, destination (icône épingle), dates début → fin (icône calendrier)
  • FAB "+" : ouvre un dialogue de création avec nom (obligatoire), destination, date de départ, date de retour
  • État vide : illustration avion + message d'invitation
  • Clic sur une carte → bascule en vue détail

Vue détail — Valise virtuelle

  • Barre supérieure : retour, nom du voyage, destination, dates, icône poubelle (suppression avec confirmation)
  • Compteur : chip "N article(s)" mis à jour en temps réel
  • Grille 3 colonnes : articles emballés en ratio 3:4, chaque carte affiche un bouton ✕ pour retirer l'article
  • État vide : icône valise + message d'invitation
  • FAB étendu "Ajouter des articles" : ouvre un bottom sheet (ModalBottomSheet) avec toute la garde-robe en grille 3 colonnes — tap sur un article l'ajoute (bordure or + overlay ✓) ou le retire s'il est déjà présent
  • Retour système (bouton physique) : revient à la liste sans perdre la sélection

E-06 — Statistiques (Style Insights)

Tableau de bord analytics de la garde-robe.

  • KPI cards : total tenues générées + articles jamais portés
  • Répartition par catégorie : liste avec barre de progression proportionnelle et compteur

E-07 — Paramètres

Configuration personnelle de l'utilisateur.

  • Avatar : cercle 80dp cliquable, ouvre le sélecteur photo, badge "✏" en surimpression
  • Badge premium : statut membre affiché
  • Mode sombre : interrupteur toggle
  • Style culturel : chips horizontaux sélectionnables
  • Langue : grille 4 colonnes de drapeaux + nom natif (8 langues)
  • Déconnexion : bouton avec bordure rouge

Accessibilité

  • Support RTL natif pour l'arabe (layout mirroring)
  • Taille minimale des zones tactiles : 48×48 dp
  • Contraste couleur conforme WCAG AA (ratio ≥ 4,5:1 pour le texte)
  • Labels contentDescription sur tous les éléments interactifs
  • Compatible TalkBack (lecteur d'écran Android)

Compatibilité

  • Android minimum : API 26 (Android 8.0 Oreo)
  • Android cible : API 35 (Android 15)
  • Résolutions testées : 360dp → 428dp de largeur
  • Support des thèmes clair et sombre

Sécurité et données

🔒
Données 100 % locales en V1 Toutes les données (photos, articles, paramètres) sont stockées uniquement sur l'appareil. Aucune donnée personnelle n'est transmise à des serveurs tiers sauf l'appel météo (Open-Meteo, API publique sans authentification).
  • Photos copiées dans le répertoire interne de l'application (non accessibles depuis d'autres apps)
  • Base de données SQLite chiffrée via SQLDelight
  • Aucune permission réseau sauf pour l'API météo

Fiabilité

  • Fonctionnement hors-ligne complet (hors bandeau météo)
  • Aucune perte de données en cas de fermeture forcée de l'application
  • Écrans de gestion d'état vide dans tous les cas (loading / vide / erreur)

Architecture

  • Pattern : Clean Architecture — Data / Domain / Presentation
  • Module partagé : ViewModels, UseCases, Repositories, modèles de données
  • Module Android : UI Compose uniquement, sans logique métier
  • Navigation : Jetpack Navigation Compose avec état de la pile préservé
  • Injection de dépendances : Koin (singleton SettingsViewModel partagé entre écrans)

API externe

🌤️
Open-Meteo (open-meteo.com) API météo gratuite, sans clé d'authentification. Données : température, code météo, humidité. Localisation via coordonnées GPS de l'appareil. En cas d'échec réseau, le bandeau météo est masqué sans bloquer l'application.

Stockage des photos

Les photos sélectionnées depuis la galerie sont copiées dans le répertoire privé de l'application via la fonction copyImageToInternalStorage(). L'URI d'origine n'est pas conservée. Le chemin interne est stocké dans la base de données.

Partage de fichiers — FileProvider

Le partage de tenues (F-16) expose les photos via androidx.core.content.FileProvider avec l'autorité {packageName}.fileprovider. Les URIs générées sont accordées en lecture temporaire à l'application cible (flag FLAG_GRANT_READ_URI_PERMISSION). Aucune permission WRITE_EXTERNAL_STORAGE n'est requise.

Persistance des paramètres

Les préférences utilisateur (photo de profil) sont persistées via l'interface SettingsStorage définie dans le module partagé. L'implémentation Android utilise SharedPreferences (fichier closetmixer_prefs). Ce patron permet une future implémentation iOS (NSUserDefaults) sans modifier le code partagé.

L'interface s'inspire des applications de mode haut de gamme. L'esthétique repose sur la sobriété, les espaces généreux et les contrastes chauds.
