Contexte et enjeux

Le marché de la mode représente plus de 1 700 milliards de dollars mondialement. Pourtant, une étude McKinsey révèle que 60 % des vêtements achetés ne sont jamais portés. Le manque de visibilité sur sa garde-robe, la difficulté à créer des tenues cohérentes et l'absence d'organisation au quotidien engendrent achats superflus, gaspillage et frustration.

ClosetMixer répond à ce constat en proposant un assistant vestimentaire personnel numérique : photographier ses articles, générer des tenues adaptées à la météo et à l'occasion, planifier ses looks dans un calendrier et préparer ses valises de voyage.

🌍
Dimension multiculturelle différenciante L'application intègre nativement des styles culturels variés (mode modeste, K-Fashion, J-Fashion, traditionnel) et supporte 8 langues dont l'arabe (RTL). Ce positionnement vise une audience internationale souvent ignorée par les acteurs du marché.

Enjeux clés

  • Enjeu usage : simplifier la prise de décision vestimentaire quotidienne
  • Enjeu consommation : réduire les achats inutiles en valorisant l'existant
  • Enjeu culturel : proposer une application inclusive pour toutes les identités vestimentaires
  • Enjeu business : constituer une base utilisateurs qualifiée pour de futures fonctionnalités premium

Périmètre fonctionnel

L'application est organisée en 5 modules fonctionnels accessibles depuis une barre de navigation inférieure (5 onglets), complétée par les écrans Paramètres et Ajout d'article accessibles depuis la garde-robe.

👗
Garde-robe
Catalogue photographique de tous les articles, filtrage par catégorie, barre de recherche, ajout et gestion des favoris.
✨
Tenues
Génération automatique de tenues selon la météo, l'occasion et le style culturel. Partage natif Android.
📅
Calendrier
Planification des tenues en vue mensuelle ou hebdomadaire (7 colonnes), avec indicateurs visuels.
✈️
Voyage
Création de voyages avec destination et dates, valise virtuelle avec ajout/retrait d'articles.
📊
Statistiques
Tableau de bord : répartition par catégorie, articles jamais portés, total tenues générées.
⚙️
Paramètres
Photo et nom de profil, genre, notification quotidienne configurable, mode sombre, style culturel, langue.

05

Description détaillée des écrans

E-01 — Onboarding

Écran de premier lancement affiché une seule fois (flag SharedPreferences onboarding_done). Présente l'application en 5 pages swipeables.

  • Pages : Bienvenue, Cataloguez, Tenue du jour, Voyages, Partagez
  • Navigation : HorizontalPager avec dots animés (actif 24dp × 8dp, inactif 8dp × 8dp)
  • Actions : bouton "Ignorer" (toutes pages sauf dernière) + bouton "SUIVANT"/"COMMENCER"
  • Thème : fond sombre (#1A1C1B), icônes or (#E9C349), fond des cercles icône or pâle (#574500)

E-02 — Garde-robe

Écran d'accueil principal affiché après l'onboarding. Catalogue photographique complet de la garde-robe.

  • En-tête : photo de profil cliquable → Paramètres + titre "Closet Mixer" (serif doré) + bouton ⚙ + bouton ✦ Ajouter
  • Barre de recherche : OutlinedTextField avec icône loupe et icône ✕ (effacement), debounce 300 ms, recherche sur sous-catégorie, couleur et tags (F-24)
  • Chips de filtre : "Tout", Vêtement, Chaussure, Bijou, Maquillage, Couvre-chef, Accessoire, Skincare — scroll horizontal
  • Grille 2 colonnes : cartes article ratio 3:4, bouton favori (♥) en superposition, étiquette sous-catégorie + couleur
  • Suppression : appui long → dialogue de confirmation → suppression définitive (F-18)
  • État vide : message adapté selon contexte (garde-robe vide ou aucun résultat de recherche)

E-03 — Ajouter un article

Formulaire d'ajout accessible depuis le bouton "+" de la garde-robe.

  • Photo : zone cliquable 240dp → galerie système → recadrage automatique 3:4 via UCrop (ratio fixe, max 600×800, qualité 85 %) (F-23)
  • Catégorie : liste déroulante ExposedDropdownMenuBox (7 catégories)
  • Sous-catégorie : liste filtrée dynamiquement selon catégorie choisie
  • Couleur : palette de 28 pastilles colorées avec validation par coche
  • Métal : visible uniquement si catégorie = Bijou (Or, Argent, Rosé, Fantaisie, Aucun)
  • Style culturel : liste déroulante (Modest, K-Fashion, J-Fashion, Traditionnel, Neutre)
  • Bouton Enregistrer : activé uniquement si sous-catégorie renseignée

E-04 — Tenues (Génération)

Écran de génération de tenues combinant météo locale, style culturel et occasion.

  • Bandeau météo : température + description + emoji météo, fond or translucide (masqué si pas de réseau)
  • Chips style culturel : Neutre, Modeste, K-Fashion, J-Fashion, Traditionnel (scroll horizontal)
  • Chips occasion : Travail, Casual, Soirée, Voyage, Sport
  • Grille "Today's Mix" : haut (7/12) + chaussure/bijou empilés (5/12), bas pleine largeur en dessous
  • Auto-génération : tenue générée automatiquement à l'ouverture (F-20)
  • Logique météo : T° < 10°C → privilégier bottes/baskets ; style Modest → inclure couvre-chef
  • Harmonie métal : tons chauds dans tenue → or/rosé ; tons froids → argent (F-13)
  • Label bijou enrichi : sous-catégorie + métal (ex : "COLLIER · OR")
  • Partage : bouton partager → Intent natif Android (photos + description + hashtags) via FileProvider (F-16)

E-05 — Calendrier

Planification des tenues avec deux modes de vue commutables.

  • Toggle Mois/Semaine : deux segments, bascule entre les deux vues
  • Vue mois : grille 7 colonnes (Lun–Dim), décalage automatique du 1er du mois, point doré sur jours planifiés, fond or pâle sur jour sélectionné, couleur primaire pour aujourd'hui
  • Vue semaine : LazyRow de 7 colonnes (72dp chacune), miniature article ratio 3:4 ou icône "+" si vide, nom et numéro du jour, navigation semaine précédente/suivante, label de la plage de dates (F-25)
  • Navigation : flèches ← → adaptées au mode actif (mois ou semaine)
  • Panneau jour sélectionné : affiche la tenue planifiée ou invite à en assigner une
  • Bottom sheet assignation : tenues sauvegardées + génération rapide d'une nouvelle tenue

E-06 — Voyages

Gestion des déplacements et des valises virtuelles en deux vues.

Vue liste

  • Cartes voyage : nom, destination (icône épingle), dates début → fin (icône calendrier)
  • FAB "+" → dialogue de création (nom obligatoire, destination, date départ, date retour)
  • État vide : illustration avion + message d'invitation

Vue détail — Valise virtuelle

  • Barre supérieure : retour, nom du voyage, destination, dates, icône poubelle (suppression avec confirmation)
  • Compteur "N article(s)" mis à jour en temps réel
  • Grille 3 colonnes : articles en ratio 3:4, bouton ✕ pour retirer
  • FAB étendu "Ajouter des articles" → ModalBottomSheet avec garde-robe complète, tap = ajout (bordure or + overlay ✓) ou retrait si déjà présent

E-07 — Statistiques (Style Insights)

Tableau de bord analytics de la garde-robe.

  • KPI cards : total tenues générées + articles jamais portés
  • Répartition par catégorie : liste avec barre de progression proportionnelle et compteur

E-08 — Paramètres

Configuration personnelle de l'utilisateur.

  • En-tête profil : avatar circulaire 80dp cliquable (sélecteur photo, badge ✏), nom de profil affiché dynamiquement (F-27), badge "MEMBRE PREMIUM"
  • Carte Premium : statut actif avec icône étoile et chevron
  • Notification tenue du jour : toggle activer/désactiver (demande permission POST_NOTIFICATIONS sur API 33+), ligne heure configurable affichée uniquement si activée → ouvre TimePickerDialog (F-22)
  • Mode sombre : toggle persistant
  • Nom de profil : OutlinedTextField avec icône Personne, sauvegarde instantanée (F-27)
  • Genre : 3 chips Femme ♀ / Homme ♂ / Autre ⚧ avec fond or pâle sur sélection (F-26)
  • Style culturel : chips horizontaux sélectionnables (5 styles)
  • Langue : grille 4 colonnes de drapeaux + nom natif (8 langues, commutation via AppCompatDelegate)
  • Noter l'application : carte cliquable → Google Play Review API avec fallback Play Store
  • Déconnexion : bouton avec bordure rouge

Accessibilité

  • Support RTL natif pour l'arabe (layout mirroring via AppCompatDelegate)
  • Taille minimale des zones tactiles : 48×48 dp
  • Contraste couleur conforme WCAG AA (ratio ≥ 4,5:1 pour le texte)
  • Labels contentDescription sur tous les éléments interactifs
  • Compatible TalkBack (lecteur d'écran Android)

Compatibilité

  • Android minimum : API 26 (Android 8.0 Oreo)
  • Android cible : API 35 (Android 15)
  • Canal de notification créé dynamiquement pour API ≥ 26
  • Permission POST_NOTIFICATIONS demandée au runtime pour API ≥ 33
  • Résolutions testées : 360dp → 428dp de largeur
  • Support des thèmes clair et sombre

Sécurité et données

🔒
Données 100 % locales en V1 Toutes les données (photos, articles, paramètres, calendrier, voyages) sont stockées uniquement sur l'appareil. Aucune donnée personnelle n'est transmise à des serveurs tiers, sauf l'appel météo vers Open-Meteo (API publique sans authentification).
  • Photos copiées dans le répertoire interne (filesDir/articles/) — inaccessibles depuis d'autres apps
  • Fichiers UCrop temporaires dans cacheDir, effacés après copie
  • Partage de fichiers via FileProvider avec flag FLAG_GRANT_READ_URI_PERMISSION temporaire
  • Aucune permission WRITE_EXTERNAL_STORAGE requise

Fiabilité

  • Fonctionnement hors-ligne complet (hors bandeau météo)
  • WorkManager (notifications) reschedulé automatiquement après redémarrage via BootReceiver
  • Aucune perte de données en cas de fermeture forcée de l'application
  • Écrans de gestion d'état vide dans tous les cas (loading / vide / erreur)

Architecture

  • Pattern : Clean Architecture — Data / Domain / Presentation
  • Module partagé (shared/commonMain) : ViewModels, UseCases, Repositories, modèles de domaine, interface SettingsStorage
  • Module Android (androidApp) : UI Compose (écrans, composants, navigation), Workers (OutfitReminderWorker, BootReceiver), AndroidSettingsStorage
  • Module iOS (iosMain) : stubs DatabaseDriverFactory + IosModule — code partagé prêt, déploiement prévu V2
  • Navigation : Jetpack Navigation Compose avec état de la pile préservé (saveState/restoreState)
  • Injection de dépendances : Koin (ViewModels en singleton partagé entre écrans)
  • Tâches de fond : WorkManager PeriodicWorkRequest (1 jour d'intervalle), politique KEEP au démarrage / UPDATE au changement d'heure utilisateur

Persistance des paramètres (SettingsStorage)

L'interface SettingsStorage définie dans le module partagé expose les méthodes suivantes, implémentées côté Android via SharedPreferences (fichier closetmixer_prefs) :

  • saveProfilePhoto / loadProfilePhoto — chemin local de la photo
  • saveProfileName / loadProfileName — nom de profil saisi
  • saveLanguage / loadLanguage — code langue (fr, en, ar…)
  • saveGender / loadGender — clé genre (femme, homme, autre)
  • saveNotificationEnabled / loadNotificationEnabled
  • saveNotificationHour / loadNotificationHour
  • saveNotificationMinute / loadNotificationMinute

Ce patron d'interface permet une future implémentation iOS (NSUserDefaults) sans modifier le code partagé.

API externe

🌤️
Open-Meteo (open-meteo.com) API météo gratuite, sans clé d'authentification. Données récupérées : température actuelle, code WMO météo. Localisation via GPS de l'appareil (LocationManager natif Android). En cas d'échec réseau, le bandeau météo est masqué sans bloquer l'application.

Permissions Android déclarées

  • INTERNET — appel API météo
  • ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION — localisation GPS pour météo
  • POST_NOTIFICATIONS — notification tenue du jour (runtime permission API 33+)
  • RECEIVE_BOOT_COMPLETED — reschedule WorkManager après redémarrage
  • SCHEDULE_EXACT_ALARM — alarme exacte pour l'heure configurée
